## Phase 0: 프로젝트 초기화

사용자의 프로젝트 아이디어를 분석하고 기술 스택과 코딩 규칙을 결정합니다.

### 담당
**Knowledge Engine (Gemini 3)** - 광범위한 기술 스택 지식 활용

### 작업
1. 프로젝트 설명을 분석하여 적합한 기술 스택을 추천합니다.
2. 사용자의 선호도를 파악하기 위한 질문을 합니다.
3. TECH_STACK.md와 RULES.md를 생성합니다.

### 질문 항목
- **프로젝트 유형**: backend, frontend, fullstack, cli, library 등
- **언어**: Python, TypeScript, Java, Go, Rust 등
- **프레임워크**: FastAPI, Next.js, Spring Boot 등
- **코딩 스타일**: 함수형, OOP, 혼합 등
- **테스트 전략**: TDD, 통합 테스트 위주, E2E 등
- **배포 환경**: Docker, Kubernetes, Serverless 등

### TECH_STACK.md 필수 항목
```markdown
# 기술 스택

## 언어
- {language} {version}

## 프레임워크
- {framework} {version}

## 주요 라이브러리
| 라이브러리 | 버전 | 용도 |
|-----------|-----|------|
| ... | ... | ... |

## 개발 도구
- 패키지 관리자: {tool}
- 린터: {tool}
- 포맷터: {tool}
- 테스트: {tool}

## 런타임 환경
- Node/Python 버전: {version}
- 컨테이너: {option}
```

### RULES.md 필수 항목
```markdown
# 코딩 규칙

## 네이밍 컨벤션
- 파일명: {convention}
- 함수/메서드: {convention}
- 클래스: {convention}
- 상수: {convention}
- 변수: {convention}

## 코드 스타일
- 최대 줄 길이: {number}
- 들여쓰기: {spaces/tabs}
- 문자열: {single/double quotes}
- 함수형/OOP 선호도: {preference}

## 타입
- 타입 힌트: {required/optional}
- strict 모드: {yes/no}

## 에러 처리
- 패턴: {pattern}
- 커스텀 예외: {convention}

## Import 규칙
- 정렬 순서: {order}
- 그룹핑: {groups}

## 문서화
- 독스트링: {style}
- 주석 언어: {language}
```

### 출력 형식
각 문서를 마크다운 코드 블록으로 제공하세요:

```markdown:TECH_STACK.md
(내용)
```

```markdown:RULES.md
(내용)
```
