## Phase 1: 기획

사용자의 요구사항을 PRD와 User Story로 구체화합니다.

### 담당
**Knowledge Engine (Gemini 3)** + **Main Agent (Claude 4.5)** 협업
- Gemini: 시장 분석, 기술 트렌드, 유사 서비스 참조
- Claude: 요구사항 구조화, 문서 작성

### 컨텍스트
{context}

### 작업
1. 핵심 기능과 목표를 파악합니다.
2. 대상 사용자를 정의합니다.
3. MVP 범위를 결정합니다.
4. PRD.md와 USER_STORIES.md를 생성합니다.

### 질문 항목
- 이 서비스의 **핵심 가치**는 무엇인가요?
- **주요 사용자**는 누구인가요?
- **MVP**에 반드시 포함되어야 할 기능은?
- **제외해도 되는** 기능은?
- **성공 지표**는 무엇인가요?

### PRD.md 필수 항목
```markdown
# Product Requirements Document

## 1. 개요
### 1.1 프로젝트 이름
### 1.2 프로젝트 설명
### 1.3 목표 및 비전

## 2. 대상 사용자
### 2.1 페르소나
### 2.2 사용자 니즈

## 3. 핵심 기능
### 3.1 필수 기능 (MVP)
| 기능 | 설명 | 우선순위 |
|-----|------|---------|
| ... | ... | P0/P1/P2 |

### 3.2 추가 기능 (Post-MVP)

## 4. 비기능 요구사항
### 4.1 성능
### 4.2 보안
### 4.3 확장성

## 5. 제약 조건
### 5.1 기술적 제약
### 5.2 일정 제약

## 6. 성공 지표
### 6.1 KPI
### 6.2 OKR
```

### USER_STORIES.md 필수 항목
```markdown
# User Stories

## 형식
- ID: US-XXX
- 제목: [사용자]는 [기능]을 [목적]을 위해 원한다
- 수락 기준 (Acceptance Criteria)
- 우선순위: Must/Should/Could/Won't

## 스토리 목록

### US-001: 제목
**As a** [사용자 유형]
**I want to** [원하는 기능]
**So that** [달성하려는 목표]

**Acceptance Criteria:**
- [ ] 기준 1
- [ ] 기준 2

**Priority:** Must

---
(더 많은 스토리...)
```

### 출력 형식
```markdown:PRD.md
(내용)
```

```markdown:USER_STORIES.md
(내용)
```
